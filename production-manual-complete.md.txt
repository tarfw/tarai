## 1. Architecture Overview

### 1.1 Three-Mode Architecture

```
┌─────────────────────────────────────────────────────────────┐
│                    MOBILE APP (Expo)                        │
│                                                              │
│  ┌────────────────────────────────────────────────────────┐ │

│  │                                                         │ │
│  │  Database: pos.db                                      │ │
│  │  Sync: Bidirectional (1 second interval)              │ │
│  │                                                         │ │

│  │  ├── products (provider's products only)               │ │
```

### 1.2 Key Design Decisions

**1. POS Queries Local Database**
```typescript
// Counter POS - LOCAL query (fast!)
const products = await db.getAllAsync(`
  SELECT * FROM products
  WHERE providerid = ? AND available = 1
`, [myProviderId]);

// Returns: Only MY products
// Speed: <5ms
// Offline: Works perfectly
```

**2. Customer Search Queries Cloud**
```typescript
// Customer app - CLOUD search (universal)
const response = await fetch(`${API_URL}/api/search`, {
  method: 'POST',
  body: JSON.stringify({ query: 'biryani near me' })
});

// Returns: ALL providers' products
// Speed: ~200ms (network)
// Offline: Shows cached results
```

**3. Pages Concept (Universal Content Storage)**
```
"pages" = Large content blobs
├── Product descriptions (1-10KB)
├── Blog posts (5-50KB)
├── Documentation (10-100KB)
├── Articles (5-20KB)
└── Rich content (HTML/Markdown)

Storage: Cloudflare R2 ($0.015/GB)
Fetch: On-demand only
Format: {type}/{id}/page.txt
```

## 2. Core Concepts

### 2.1 Provider Data Isolation

**Each Provider Has:**
- Own products (synced to their devices)
- Own inventory (synced to their devices)
- Own embeddings (for local search)

**Example:**
```
Provider A: "Pizza Palace"
├── 50 products (Pizza varieties)
├── 100 inventory items (sizes, toppings)
├── 50 embeddings
└── Local DB size: 5 MB

Provider B: "Burger King"
├── 80 products (Burgers, sides)
├── 150 inventory items
├── 80 embeddings
└── Local DB size: 8 MB

Each device only syncs ITS provider's data!
```

**Filter Configuration:**
```typescript
// Turso sync filters by providerid
<SQLiteProvider
  options={{
    libSQLOptions: {
      url: TURSO_URL,
      authToken: TURSO_TOKEN,
      syncFilter: {
        products: `providerid = '${myProviderId}'`,
        inventoryitems: `providerid = '${myProviderId}'`
      }
    }
  }}
/>

// Result: Only sync MY data, not everyone's!
```

### 2.2 Pages Architecture

**What are Pages?**
- Large text content (>500 bytes)
- Fetched on-demand
- Not synced to mobile
- Stored in R2 (cheap!)

**Page Types:**
```typescript
type PageType = 
  | 'productdescription'
  | 'blogpost'
  | 'documentation'
  | 'article'
  | 'help'
  | 'terms'
  | 'privacy';

interface Page {
  pageid: string;
  type: PageType;
  entityid: string;  // productid, postid, etc.
  content: string;   // Full text/HTML/Markdown
  format: 'text' | 'html' | 'markdown';
  created: number;
  updated: number;
}
```

**Storage Structure:**
```
R2 Bucket: pages/
├── productdescription/
│   ├── p123/page.txt
│   ├── p456/page.html
│   └── p789/page.md
├── blogpost/
│   ├── post1/page.md
│   └── post2/page.html
└── documentation/
    ├── getting-started/page.md
    └── api-reference/page.html
```

### 2.3 Boolean Flag Strategy

**All binary states use INTEGER (0 or 1):**
```sql
available INTEGER DEFAULT 1     -- Product available
instock INTEGER                 -- Has stock
paid INTEGER DEFAULT 0          -- Payment received
completed INTEGER DEFAULT 0     -- Order complete
active INTEGER DEFAULT 1        -- Account active
online INTEGER DEFAULT 0        -- User online
done INTEGER DEFAULT 0          -- Task complete
featured INTEGER DEFAULT 0      -- Featured item
verified INTEGER DEFAULT 0      -- Verified user
```

**Benefits:**
- 1 byte per flag (vs 4-20 bytes for strings)
- Fast indexed queries
- Clear semantics
- 75-95% storage savings

## 3. Database Design

### 3.1 Turso Schema (Synced Data)

**Per-Provider Filtered Sync**

```sql
-- ================================================
-- PRODUCTS (Synced per provider)
-- ================================================
CREATE TABLE products (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  category TEXT NOT NULL,
  subcategory TEXT NOT NULL,
  baseprice REAL NOT NULL,
  currency TEXT DEFAULT 'INR',
  
  -- NO PAGE CONTENT HERE!
  haspage INTEGER DEFAULT 0,  -- Flag: has description page
  
  -- Location
  lat REAL,
  lng REAL,
  address TEXT,
  serviceradius REAL,
  
  -- Provider (CRITICAL for filtering)
  providerid TEXT NOT NULL,
  providername TEXT,
  
  -- Timing
  preptime INTEGER,
  
  -- Metadata (small, <500 bytes)
  meta TEXT,
  
  -- Search keywords (short)
  searchtext TEXT,
  
  -- FLAGS
  available INTEGER DEFAULT 1,
  hasvariants INTEGER DEFAULT 0,
  featured INTEGER DEFAULT 0,
  
  -- Timestamps
  created INTEGER NOT NULL,
  updated INTEGER NOT NULL
);

CREATE INDEX idx_products_provider ON products(providerid);
CREATE INDEX idx_products_category ON products(category);
CREATE INDEX idx_products_available ON products(available) WHERE available = 1;

-- Size: ~150 bytes per product
-- 1000 products per provider = 150 KB

-- ================================================
-- INVENTORY ITEMS (Real-time sync critical!)
-- ================================================
CREATE TABLE inventoryitems (
  id TEXT PRIMARY KEY,
  productid TEXT NOT NULL,
  providerid TEXT NOT NULL,  -- For filtering
  variantname TEXT,
  
  -- Stock (MUST sync across all devices!)
  quantity INTEGER DEFAULT 0,
  reserved INTEGER DEFAULT 0,
  
  -- Pricing
  price REAL NOT NULL,
  cost REAL,
  
  -- Alerts
  minstock INTEGER DEFAULT 0,
  maxstock INTEGER,
  
  -- FLAGS (computed)
  instock INTEGER AS (quantity > 0),
  lowstock INTEGER AS (quantity <= minstock),
  
  -- Timestamps
  created INTEGER NOT NULL,
  updated INTEGER NOT NULL,
  
  FOREIGN KEY (productid) REFERENCES products(id)
);

CREATE INDEX idx_inventory_provider ON inventoryitems(providerid);
CREATE INDEX idx_inventory_product ON inventoryitems(productid);
CREATE INDEX idx_inventory_instock ON inventoryitems(instock) WHERE instock = 1;

-- Size: ~80 bytes per item
-- Sync interval: 1 second (critical!)

-- ================================================
-- EMBEDDINGS (For local semantic search)
-- ================================================
CREATE TABLE embeddings (
  productid TEXT PRIMARY KEY,
  providerid TEXT NOT NULL,  -- For filtering
  embedding F32_BLOB(768),
  created INTEGER NOT NULL,
  FOREIGN KEY (productid) REFERENCES products(id)
);

CREATE INDEX idx_embeddings_provider ON embeddings(providerid);
CREATE INDEX idx_embeddings_vector ON embeddings(embedding);

-- Size: ~3KB per embedding
-- 1000 products = 3 MB

-- ================================================
-- PROVIDERS
-- ================================================
CREATE TABLE providers (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  type TEXT NOT NULL,
  lat REAL,
  lng REAL,
  address TEXT,
  phone TEXT,
  meta TEXT,
  active INTEGER DEFAULT 1,
  created INTEGER NOT NULL
);

-- ================================================
-- FTS (Full-text search - local)
-- ================================================
CREATE VIRTUAL TABLE productsfts USING fts5(
  productid UNINDEXED,
  providerid UNINDEXED,
  name,
  searchtext
);

-- Query with provider filter:
-- SELECT * FROM productsfts WHERE productsfts MATCH 'biryani'
-- AND providerid = ?
```

### 3.2 Local-Only Tables (Not Synced)

```sql
-- ================================================
-- AGENT MEMORY (VoltAgent storage)
-- ================================================
CREATE TABLE agentmemory (
  id TEXT PRIMARY KEY,
  conversationid TEXT NOT NULL,
  agentname TEXT NOT NULL,
  
  -- Content
  role TEXT NOT NULL,
  content TEXT NOT NULL,
  
  -- Type
  memorytype TEXT,
  importance REAL DEFAULT 0.5,
  
  -- References
  productid TEXT,
  orderid TEXT,
  userid TEXT,
  
  -- Semantic search
  embedding F32_BLOB(768),
  
  -- Timestamps
  created INTEGER NOT NULL,
  accessed INTEGER NOT NULL,
  expires INTEGER
);

CREATE INDEX idx_memory_conversation ON agentmemory(conversationid);
CREATE INDEX idx_memory_agent ON agentmemory(agentname);
CREATE INDEX idx_memory_embedding ON agentmemory(embedding);

-- Private per device, never syncs

-- ================================================
-- CART (Session data)
-- ================================================
CREATE TABLE cart (
  id TEXT PRIMARY KEY,
  userid TEXT NOT NULL,
  sessionid TEXT NOT NULL,
  items TEXT NOT NULL,  -- JSON
  subtotal REAL,
  tax REAL,
  discount REAL,
  total REAL,
  created INTEGER NOT NULL,
  updated INTEGER NOT NULL
);

CREATE INDEX idx_cart_session ON cart(sessionid);

-- ================================================
-- DRAFT ORDERS (Temporary)
-- ================================================
CREATE TABLE draftorders (
  id TEXT PRIMARY KEY,
  ordernumber TEXT UNIQUE NOT NULL,
  userid TEXT NOT NULL,
  providerid TEXT NOT NULL,
  items TEXT NOT NULL,  -- JSON
  subtotal REAL NOT NULL,
  tax REAL,
  discount REAL,
  total REAL NOT NULL,
  paid INTEGER DEFAULT 0,
  created INTEGER NOT NULL,
  updated INTEGER NOT NULL
);

CREATE INDEX idx_drafts_provider ON draftorders(providerid);

-- Deleted after order completion
```

### 5.3 Root Layout (Database Providers)

```typescript
// app/_layout.tsx
import { useEffect, useState } from 'react';
import { SQLiteProvider, useSQLiteContext } from 'expo-sqlite';
import { Stack } from 'expo-router';
import { QueryClient, QueryClientProvider } from '@tanstack/react-query';
import { initDatabase } from '../src/db/init';
import { AuthProvider } from '../src/contexts/AuthContext';

const queryClient = new QueryClient();

export default function RootLayout() {
  const [dbInitialized, setDbInitialized] = useState(false);
  
  return (
    <SQLiteProvider
      databaseName="pos.db"
      onInit={async (db) => {
        await initDatabase(db);
        setDbInitialized(true);
      }}
      options={{
        // Turso embedded replica (official method)
        libSQLOptions: {
          url: process.env.EXPO_PUBLIC_TURSO_URL!,
          authToken: process.env.EXPO_PUBLIC_TURSO_TOKEN!,
          
          // Real-time sync for inventory
          syncInterval: 1000,  // 1 second
          syncMode: 'bidirectional',
          
          // Provider-specific filter
          syncFilter: {
            products: `providerid = '${process.env.EXPO_PUBLIC_PROVIDER_ID}'`,
            inventoryitems: `providerid = '${process.env.EXPO_PUBLIC_PROVIDER_ID}'`,
            embeddings: `providerid = '${process.env.EXPO_PUBLIC_PROVIDER_ID}'`,
          },
          
          // Connection
          maxRetries: 5,
          retryInterval: 1000,
        },
      }}
    >
      <QueryClientProvider client={queryClient}>
        <AuthProvider>
          {dbInitialized && <Stack />}
        </AuthProvider>
      </QueryClientProvider>
    </SQLiteProvider>
  );
}
```

### 5.4 Database Initialization

```typescript
// src/db/init.ts
import type { SQLiteDatabase } from 'expo-sqlite';

export async function initDatabase(db: SQLiteDatabase) {
  console.log('Initializing local-only tables...');
  
  try {
    // Agent Memory (Local only)
    await db.execAsync(`
      CREATE TABLE IF NOT EXISTS agentmemory (
        id TEXT PRIMARY KEY,
        conversationid TEXT NOT NULL,
        agentname TEXT NOT NULL,
        role TEXT NOT NULL,
        content TEXT NOT NULL,
        memorytype TEXT,
        importance REAL DEFAULT 0.5,
        productid TEXT,
        orderid TEXT,
        userid TEXT,
        embedding F32_BLOB(768),
        created INTEGER NOT NULL,
        accessed INTEGER NOT NULL,
        expires INTEGER
      );
      
      CREATE INDEX IF NOT EXISTS idx_memory_conversation 
        ON agentmemory(conversationid);
      CREATE INDEX IF NOT EXISTS idx_memory_agent 
        ON agentmemory(agentname);
      CREATE INDEX IF NOT EXISTS idx_memory_embedding 
        ON agentmemory(embedding);
    `);
    
    // Cart (Local only)
    await db.execAsync(`
      CREATE TABLE IF NOT EXISTS cart (
        id TEXT PRIMARY KEY,
        userid TEXT NOT NULL,
        sessionid TEXT NOT NULL,
        items TEXT NOT NULL,
        subtotal REAL,
        tax REAL,
        discount REAL,
        total REAL,
        created INTEGER NOT NULL,
        updated INTEGER NOT NULL
      );
      
      CREATE INDEX IF NOT EXISTS idx_cart_session ON cart(sessionid);
    `);
    
    // Draft Orders (Local only)
    await db.execAsync(`
      CREATE TABLE IF NOT EXISTS draftorders (
        id TEXT PRIMARY KEY,
        ordernumber TEXT UNIQUE NOT NULL,
        userid TEXT NOT NULL,
        providerid TEXT NOT NULL,
        items TEXT NOT NULL,
        subtotal REAL NOT NULL,
        tax REAL,
        discount REAL,
        total REAL NOT NULL,
        paid INTEGER DEFAULT 0,
        created INTEGER NOT NULL,
        updated INTEGER NOT NULL
      );
      
      CREATE INDEX IF NOT EXISTS idx_drafts_provider 
        ON draftorders(providerid);
    `);
    
    console.log('Local tables created successfully');
    
    // Wait for initial sync
    console.log('Waiting for initial sync from Turso...');
    await new Promise(resolve => setTimeout(resolve, 2000));
    
    // Verify synced data
    const products = await db.getAllAsync('SELECT COUNT(*) as count FROM products');
    const inventory = await db.getAllAsync('SELECT COUNT(*) as count FROM inventoryitems');
    
    console.log(`Synced: ${products[0].count} products, ${inventory[0].count} inventory items`);
    
  } catch (error) {
    console.error('Database initialization error:', error);
    throw error;
  }
}
```

### 5.6 Database Queries (Local POS)

```typescript
// src/db/queries.ts
import { useSQLiteContext } from 'expo-sqlite';

// Hook for database access
export function useDatabase() {
  return useSQLiteContext();
}

// Get provider's products (LOCAL query)
export async function getProviderProducts(
  db: any,
  providerid: string,
  category?: string
) {
  const query = category
    ? `SELECT p.*, i.price, i.quantity, i.instock
       FROM products p
       JOIN inventoryitems i ON i.productid = p.id
       WHERE p.providerid = ? AND p.category = ? AND p.available = 1
       ORDER BY p.name`
    : `SELECT p.*, i.price, i.quantity, i.instock
       FROM products p
       JOIN inventoryitems i ON i.productid = p.id
       WHERE p.providerid = ? AND p.available = 1
       ORDER BY p.name`;
  
  const params = category ? [providerid, category] : [providerid];
  return await db.getAllAsync(query, params);
}

// Search provider's products (LOCAL semantic search)
export async function searchProviderProducts(
  db: any,
  providerid: string,
  query: string,
  embedding: Float32Array
) {
  // Vector search
  const results = await db.getAllAsync(`
    SELECT 
      p.*,
      i.price,
      i.quantity,
      i.instock,
      vector_distance_cos(e.embedding, ?) as similarity
    FROM products p
    JOIN inventoryitems i ON i.productid = p.id
    JOIN embeddings e ON e.productid = p.id
    WHERE p.providerid = ? AND p.available = 1
    HAVING similarity > 0.7
    ORDER BY similarity DESC
    LIMIT 20
  `, [embedding, providerid]);
  
  return results;
}

// Get inventory item (LOCAL)
export async function getInventoryItem(db: any, itemid: string) {

export async function updateInventory(

  quantity: number,
  reserved: number

  await db.runAsync(`
    UPDATE inventoryitems
    SET quantity = ?, reserved = ?, updated = ?

}

// Create draft order (LOCAL)
export async function createDraftOrder(
  db: any,
  order: {
    ordernumber: string;
    userid: string;
    providerid: string;
    items: any[];
    total: number;
  }
) {
  const subtotal = order.items.reduce((sum, item) => sum + item.price * item.quantity, 0);
  const tax = subtotal * 0.18;
  const discount = 0;
  
  await db.runAsync(`
    INSERT INTO draftorders (
      id, ordernumber, userid, providerid, items,
      subtotal, tax, discount, total, paid,
      created, updated

  `, [
    generateId(),
    order.ordernumber,
    order.userid,
    order.providerid,
    JSON.stringify(order.items),
    subtotal,
    tax,
    discount,
    order.total,

    Date.now(),
  ]);
}
```

```typescript
// src/services/pages.ts
const PAGES_URL = process.env.EXPO_PUBLIC_PAGES_URL;

export interface Page {
  pageid: string;
  type: string;
  entityid: string;
  content: string;
  format: 'text' | 'html' | 'markdown';
  created: number;
  updated: number;
}

export async function fetchPage(
  type: string,
  entityid: string
): Promise<Page | null> {
  try {
    // Fetch from R2
    const url = `${PAGES_URL}/${type}/${entityid}/page.txt`;
    const response = await fetch(url);
    
    if (!response.ok) {
      return null;
    }
    
    const content = await response.text();
    
    return {
      pageid: `${type}-${entityid}`,
      type,
      entityid,
      content,
      format: 'text',
      created: Date.now(),
      updated: Date.now(),
    };
  } catch (error) {
    console.error('Failed to fetch page:', error);
    return null;
  }
}

export async function createPage(
  type: string,
  entityid: string,
  content: string,
  format: 'text' | 'html' | 'markdown' = 'text'
): Promise<boolean> {
  try {
    // Upload to R2 via API
    const response = await fetch(`${API_URL}/api/pages/create`, {
      method: 'POST',
      headers: { 'Content-Type': 'application/json' },
      body: JSON.stringify({ type, entityid, content, format }),
    });
    
    return response.ok;
  } catch (error) {
    console.error('Failed to create page:', error);
    return false;
  }
}
```

## 7. API Implementation (Cloudflare Workers)

### 7.1 Product Search API

```typescript
// workers/src/api/search.ts
import { embed } from 'ai';
import { google } from '@ai-sdk/google';

export async function searchProducts(request: Request, env: Env) {
  const { query, userlat, userlng, radius, category } = await request.json();
  
  // Generate query embedding
  const { embedding: queryEmbedding } = await embed({
    model: google.textEmbeddingModel('text-embedding-004'),
    value: query,
  });
  
  // Vector search in Turso Cloud
  const results = await env.TURSO_DB.execute({
    sql: `
      SELECT 
        p.id,
        p.name,
        p.category,
        p.baseprice,
        p.haspage,
        pr.name as providername,
        pr.lat as providerlat,
        pr.lng as providerlng,
        i.price,
        i.quantity,
        i.instock,
        vector_distance_cos(e.embedding, ?) as similarity,
        (
          6371 * acos(
            cos(radians(?)) * cos(radians(pr.lat)) *
            cos(radians(pr.lng) - radians(?)) +
            sin(radians(?)) * sin(radians(pr.lat))
          )
        ) as distance
      FROM products p
      JOIN providers pr ON p.providerid = pr.id
      JOIN inventoryitems i ON i.productid = p.id
      JOIN embeddings e ON e.productid = p.id
      WHERE 
        p.available = 1
        AND i.instock = 1
        ${category ? 'AND p.category = ?' : ''}
      HAVING 
        similarity > 0.7
        AND distance <= ?
      ORDER BY 
        similarity DESC,
        distance ASC
      LIMIT 20
    `,
    args: [
      new Float32Array(queryEmbedding),
      userlat,
      userlng,
      userlat,
      ...(category ? [category] : []),
      radius,
    ],
  });
  
  return Response.json({
    products: results.rows,
  });
}
```

### 7.2 Pages API

```typescript
// workers/src/api/pages.ts
export async function createPage(request: Request, env: Env) {
  const { type, entityid, content, format } = await request.json();
  
  // Upload to R2
  const key = `${type}/${entityid}/page.${format === 'html' ? 'html' : format === 'markdown' ? 'md' : 'txt'}`;
  
  await env.PAGES_BUCKET.put(key, content, {
    httpMetadata: {
      contentType: format === 'html' ? 'text/html' : 'text/plain',
    },
  });
  
  return Response.json({ success: true, key });
}

export async function getPage(request: Request, env: Env) {
  const { type, entityid } = request.params;
  
  // Try different formats
  const formats = ['txt', 'html', 'md'];
  
  for (const format of formats) {
    const key = `${type}/${entityid}/page.${format}`;
    const object = await env.PAGES_BUCKET.get(key);
    
    if (object) {
      return new Response(object.body, {
        headers: {
          'Content-Type': object.httpMetadata.contentType,
        },
      });
    }
  }
  
  return new Response('Page not found', { status: 404 });
}
```

## 8. Data Flows

### 8.1 POS Transaction Flow

```
COUNTER POS (Provider Device)

1. Cashier searches product
   ↓
   LOCAL query: SELECT * FROM products WHERE providerid = ?
   ↓
   Results in <5ms (local SQLite)

2. Add to cart
   ↓
   LOCAL: INSERT INTO cart

3. Checkout
   ↓
   LOCAL: INSERT INTO draftorders
   ↓
   LOCAL: UPDATE inventoryitems SET quantity = quantity - ?
   ↓
   Turso syncs in background (1 second)

4. Payment complete
   ↓
   API: POST /api/orders/create
   ↓
   InstantDB: Create order + activeorder
   ↓
   LOCAL: DELETE FROM draftorders

5. Other devices
   ↓
   Receive inventory update via Turso sync (<2 seconds)
   ↓
   UI updates automatically

TOTAL TIME: ~2 seconds for full multi-device sync
```

### 8.2 Customer Discovery Flow

```
CUSTOMER APP

1. User searches "biryani near me"
   ↓
   Get location
   ↓
   API: POST /api/search
   {
     query: "biryani",
     userlat: 13.0827,
     userlng: 80.2707,
     radius: 10
   }

2. Cloud vector search
   ↓
   Turso Cloud:
   - Vector similarity
   - Distance calculation
   - Availability filter
   ↓
   Returns top 20 results from ALL providers

3. User taps product
   ↓
   Navigate to product detail
   ↓
   Fetch page (description) from R2
   ↓
   Display full product info

4. User adds to cart
   ↓
   Place order
   ↓
   Order created in InstantDB
   ↓
   Provider receives notification (real-time)

TOTAL TIME: ~500ms for search (network dependent)
```

### 8.3 Product Creation Flow

```
PROVIDER DEVICE (Admin Mode)

1. Provider creates product via API
   ↓
   Input: "Chicken Biryani, half 180, full 320"
   Input: Full description (2KB)
   ↓
   API: POST /api/products/create

2. Cloudflare Worker + VoltAgent
   ↓
   Gemma 3: Parse input
   ↓
   Extract:
   - Core product data
   - Variants (from agent memory)
   - Page content (description)

3. Store in databases
   ↓
   Turso: INSERT product (without description)
   Turso: INSERT inventoryitems (for variants)
   Turso: INSERT embedding
   R2: PUT description page
   ↓
   Sync to all provider devices (<5 seconds)

4. Provider devices
   ↓
   Turso sync downloads new product
   ↓
   Available in POS immediately
   ↓
   Description fetched only when viewing detail

TOTAL TIME: ~5 seconds for product to appear on all devices
```
